# 日本語学習チャットボット PWA 要件定義書（本番版）

> UIモックではなく、本番運用する前提のフル稼働版。
> たくみ本人のプロダクトとしてリリースする。

---

## 1. 目的

日本語学習者向けのチャットボットPWAを作る。
スマホで自然に使えて、ホーム画面追加でアプリのように使える、**実運用レベル**のサービスを作る。

デモやモックではない。本番で動くもの を最終成果物とする。

---

## 2. プロダクトの位置づけ

- LPやSNSから流入 → チャットボット画面に入る
- ログインなしでもすぐ使える（フリクションゼロ）
- スマホでそのままPWAとして追加して毎日使える
- 日本語学習の質問にフレンドリーな先生のように答える
- 将来的にCTA導線、課金、会員機能を追加する余地を残す

---

## 3. 対象ユーザー

- 日本語を学んでいる外国人（英語話者中心）
- 初級〜中級
- スマホ中心で使う
- 漢字、文法、単語、例文、日常表現などをサクッと聞きたい層

---

## 4. スコープ

### 今回やること（本番稼働に必要な最低限すべて）

- Webで使えるチャットUI（PC/スマホ両対応）
- PWA対応（manifest、service worker、ホーム画面追加）
- AIからの応答が**実際に返ってくる**（APIキー秘匿化）
- 会話履歴はセッション単位で保持（ブラウザを閉じても復元可能）
- URLで共有可能
- Supabaseでバックエンド構築（既存プロジェクトを利用）
- Vercelで本番デプロイ

### 今回やらないこと

- ログイン/会員登録（匿名ユーザーIDのみ）
- 課金機能
- 管理画面
- Push通知
- お気に入り / ブックマーク
- 学習進捗の可視化
- 複雑なオンボーディング

### 将来拡張で足す前提

- 認証（Supabase Auth）
- 課金（Stripe）
- 会話履歴の永続化（匿名→認証ユーザーへの引き継ぎ）
- CTA導線、紹介機能

---

## 5. 機能要件

### 5-1. チャット

- ユーザー発言は右寄せ、AI返答は左寄せ
- ストリーミング応答（タイピング中のようにリアルタイムで文字が出る）
- 送信中はローディング表示
- 失敗時はエラーUI + 再送ボタン
- 初回は質問サンプルを表示（「漢字の読み方」「文法チェック」「自然な例文」など）
- 長文・改行対応

### 5-2. 入力

- 下部固定の入力欄
- Enter送信（Shift+Enterで改行）
- 送信ボタン
- 複数行対応で自動拡張

### 5-3. セッション管理

- 初回アクセス時、匿名ユーザーIDを発行しlocalStorageに保存
- Supabaseに `sessions` と `messages` を保存
- ブラウザ再訪時は前回のセッションを復元
- 「新しいチャット」で新規セッション開始

### 5-4. PWA

- manifest.json
- service worker（オフライン時のシェル表示）
- ホーム画面追加の案内モーダル（iPhone向けに自然な見せ方）
- アイコン各サイズ用意
- スタンドアロン起動で違和感のない見た目

### 5-5. AI応答

- OpenAI API を利用（モデルは `gpt-4o-mini`、将来的に切り替え可能な構造）
- Next.js の API Route または Supabase Edge Function 経由で呼び出し、**APIキーはフロントに出さない**
- システムプロンプトは編集しやすい1ファイルで管理
- 応答はストリーミングで返す

### 5-6. エラー / 空状態 / ローディング

- ネットワークエラー用のカードUI
- 初回の空チャット画面はサンプル質問を並べる
- 応答待ちは「考え中」の自然なアニメーション

---

## 6. デザイン要件

### トーン

- 白ベース + 水色アクセント
- 清潔感、余白多め
- 学習サービス感
- 信頼できるが派手すぎない
- 文字が読みやすい

### 避けるもの

- 情報量過多
- カラフルすぎ
- エンジニア向けダッシュボード感
- 子供っぽすぎ

### 重視するもの

- 第一印象で用途がわかる
- 余白が整っている
- スマホで片手操作できる
- タップしやすいサイズ
- 入力したくなる見た目

## 7. 必須画面

1. **ウェルカム画面**（初回のみ / スキップ可）
   - サービス名、キャッチコピー、「始める」CTA
2. **チャット画面**（メイン）
   - ヘッダー（サービス名、メニュー、新規チャット）
   - メッセージ履歴
   - 入力欄
   - サンプル質問（初回のみ）
3. **設定画面**（シンプル）
   - PWA追加案内
   - 使い方
   - 言語切替（UI言語、英/日）
   - 会話履歴クリア
4. **PWA追加案内モーダル**
   - iPhone Safari用の手順
   - Android Chrome用の手順
   - 閉じたら一定期間再表示しない

---

## 8. 技術スタック

- **フロント**: Next.js 14+（App Router） + TypeScript + Tailwind CSS
- **UIコンポーネント**: shadcn/ui
- **バックエンド**: Supabase（既存プロジェクト利用）
  - `messages`, `sessions` テーブル
  - Edge Function で Kimi API をプロキシ（APIキー秘匿）
  - RLS で匿名ユーザーが自分のセッションのみ読み書きできるように
- **AI API**: OpenAI API（`gpt-4o-mini` を基本モデルに）
- **ホスティング**: Vercel
- **PWA**: next-pwa または自前でservice worker管理

---

## 9. データモデル（Supabase）

### `sessions`
- `id` (uuid, pk)
- `anon_user_id` (text) — localStorageで発行する匿名ID
- `title` (text, nullable) — 最初のメッセージから自動生成
- `created_at` (timestamptz)
- `updated_at` (timestamptz)

### `messages`
- `id` (uuid, pk)
- `session_id` (uuid, fk → sessions.id)
- `role` (text: "user" | "assistant")
- `content` (text)
- `created_at` (timestamptz)

### RLS
- 匿名ユーザーは自分の `anon_user_id` のセッション/メッセージのみ読み書き可

## 10. プロンプト設計

- システムプロンプトは `prompts/system.ts` などの1ファイルで管理
- 内容の骨子：
  - フレンドリーな日本語の先生として振る舞う
  - デフォルトは英語で返答、必要に応じて日本語を混ぜる
  - ユーザーが指定した言語に柔軟対応
  - 免責文は不要、自然に答える
- **あとから変更しやすい構造**が最優先

## 11. 納品条件（本番稼働レベル）

- Vercel本番URLで動く
- スマホでPWAとして追加できる
- 実際にAIが応答を返す
- 会話がSupabaseに保存される
- 再訪時に履歴が復元される
- Lighthouse PWAチェックをパスする
- 環境変数で全シークレットを管理（OpenAI API Key、Supabase Service Role Key）

---

## 12. 優先順位

### 最優先
1. チャットが実際に動く（AI応答 / ストリーミング / 保存）
2. スマホでの使用感
3. Supabase統合とAPIキー秘匿
4. PWA追加が自然にできる

### 次点
- 設定画面
- エラーUI/空状態の作り込み
- ウェルカム画面

### 後回し
- 認証
- 課金
- 詳細なオンボーディング
- 管理画面

## 13. 開発方針

- 本番想定だが、過剰設計はしない
- コンポーネント分割は後から認証や課金を差し込める粒度
