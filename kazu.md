岸田コウスケ向け 要件定義書（MVP版） 
1. 文書の目的 
この文書は、岸田コウスケが Claude Code を使って日本語学習用チャットボットのMVPを開発するための前提整理書 である。
目的は、細かく仕様を縛ることではなく、背景・目的・制約条件・優先順位を明確にして、最短で使えるものを作ること。

2. プロジェクト概要 
作りたいものは、日本語学習者向けのWebチャットボット。
ユーザーが日本語学習について質問すると、フレンドリーな先生のように答えてくれる シンプルなチャットボットを作る。

今回は高機能な完成版ではなく、まず使えるMVPを最短で作ること を優先する。

3. 背景 
依頼人は、日本語学習支援のためのチャットボットを作りたい。
ただし最初から作り込みすぎる必要はなく、まずは 簡単でも動くもの を出したい。
また、完璧主義で止まることを避け、進捗を見える化しながら前に進めることを重視する。

細かく作り込みすぎず、あとから変えやすさ を優先する。

16. 岸田コウスケの行動ルール 
岸田コウスケは、完璧主義や失敗への不安で止まりやすい前提で進める。
そのため、以下をルールとする。

完璧を目指しすぎない
止まったまま悩まない
分からないことは早めに匠君に聞く
聞くことを遠慮しない
手を動かして進めることを優先する
迷ったら小さく実装して確認する
また、匠君に相談する時は、当然と思わず感謝を持って相談すること。

17. 作業終わりの振り返りルール 
毎回の作業終わりに、必ず依頼人へ報告すること。
報告は長くなくてよい。以下の4点だけでよい。

報告フォーマット 
今日やったこと
今できていること
詰まっていること
次にやること
報告の目的 
完璧主義で止まっていないかを見える化するため
遅れているかどうかを依頼人が把握できるようにするため
進捗がゼロに見えないようにするため
小さく前進している実感を持つため
報告は、見栄を張らず、短く事実ベースでよい。

18. Claude Codeへの依頼方針 
Claude Code には、細かすぎる指示は与えすぎない。
背景、目的、制約、優先順位だけを渡し、実装方法はある程度任せる。

理由は、細かく縛りすぎると柔軟性が落ち、かえって進みにくくなるため。
特に今回はMVPなので、迷わず進めて、必要最低限の形に早く到達すること を最優先にする。
